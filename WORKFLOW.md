# 工作流

### 基本原则

1. `main` 分支不直接修改，所有实质性改动通过 Pull Request（PR）；
2. 新增稿件由技术员按各责任编辑负责的板块放入对应目录；
3. 稿件开头必须包含元数据；
4. 责任编辑可以修改自己负责的稿件。
5. 非责任编辑修改稿件时，由 CODEOWNERS 通知对应责任编辑处理。
6. 修改责编、移动稿件、修改工作流或自动化规则，只由 bypass 人员处理。
