# 工作流

### 稿件工作流程

- 技术员接收投稿；
- 技术员将稿件 Markdown 化、写上元数据并传到对应责编的文件夹；
- 责编粗修标点与错别字；
- 如果需要的话责编进行下一步的修改或提出修改建议；
- 责编与作者联系确认（建议使用腾讯文档或 QQ）；
- 责编提交会被 @YzCJ-Bot 自动批准、合并的 Pull Request（PR）；
- 稿件进入公审阶段，所有编辑可用 Issues 提出大方向建议，PR 提出具体建议；
- 责编审核并与作者再次确认后合并进 main 分支。

### 元数据范例（YAML 格式）

```
title: "文章标题"
author: "班级 作者"
author_display: "作者实际署名"
received_date: "yyyy-mm-dd"
editor: "对应责编"
editor_username: "对应责编的 GitHub 用户名"
status: "文章状态"
---
```
